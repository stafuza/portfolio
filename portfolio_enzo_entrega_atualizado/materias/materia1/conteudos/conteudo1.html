GIT e GitHub

Durante a disciplina de Projeto Integrador, aprendemos a utilizar ferramentas essenciais de controle de versão para projetos de desenvolvimento de software: o Git e o GitHub.

Por que usar Git?

Em projetos de software, é comum termos várias versões de arquivos sendo modificadas por diferentes pessoas da equipe. O Git foi criado para solucionar problemas como perda de arquivos, controle de alterações e colaboração entre desenvolvedores.

  • Permite rastrear alterações no código-fonte;
  • Facilita o trabalho em equipe por meio de branches e merges;
  • Possibilita a reversão de alterações com segurança;
  • É gratuito, open source e de alta performance.

Git vs GitHub

O Git é o sistema de controle de versão, enquanto o GitHub é uma plataforma online onde podemos armazenar nossos repositórios Git, colaborar com outras pessoas e compartilhar nossos projetos.

"Git é a ferramenta de versionamento. GitHub é o local onde seus repositórios Git podem ser hospedados, visualizados e compartilhados."

Comandos Básicos Utilizados

  • git init – Inicializa um repositório Git local.
  • git add . – Adiciona todas as alterações ao staging area.
  • git commit -m "mensagem" – Salva as alterações no histórico do repositório.
  • git status – Exibe o estado atual dos arquivos.
  • git log – Mostra o histórico de commits.
  • git push origin main – Envia as alterações para o repositório remoto no GitHub.
  • git pull origin main – Atualiza o repositório local com as alterações do GitHub.

Branches e Colaboração

Trabalhar com branches permite desenvolver novas funcionalidades sem afetar o código principal. Comandos importantes:

  • git branch nome_da_branch – Cria uma nova branch.
  • git switch nome_da_branch – Muda para a branch especificada.
  • git merge nome_da_branch – Mescla as alterações de uma branch para outra.

GitHub e o Fluxo de Trabalho

Com o GitHub, aprendemos a criar repositórios, clonar projetos, fazer push e pull das alterações, além de usar recursos como pull requests, que permitem revisões antes de mesclar alterações de diferentes desenvolvedores.

Segurança: Token de Acesso e SSH

Para garantir segurança nas conexões com o GitHub, aprendemos a gerar tokens de acesso pessoal (PAT) e configurar autenticação via chave SSH.
