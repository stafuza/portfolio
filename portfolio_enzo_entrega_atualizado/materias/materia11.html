Sobre a Matéria

A disciplina de Técnicas Avançadas de Banco de Dados Relacional e Não Relacional explora conceitos e tecnologias avançadas para armazenamento, gerenciamento e recuperação de dados em diferentes paradigmas. Ela vai além dos fundamentos básicos de bancos de dados, abordando técnicas sofisticadas para lidar com grandes volumes de dados e requisitos complexos.

Na parte relacional, são estudados tópicos como modelagem avançada, normalização e desnormalização, otimização de consultas, índices, stored procedures, triggers, transações, controle de concorrência e recuperação. Os alunos aprendem a projetar e implementar bancos de dados relacionais eficientes e escaláveis usando sistemas como MySQL, PostgreSQL e SQL Server.

Na parte não relacional (NoSQL), a disciplina aborda diferentes tipos de bancos de dados como documentos (MongoDB), colunas (Cassandra), chave-valor (Redis), grafos (Neo4j) e suas aplicações. São explorados conceitos como escalabilidade horizontal, consistência eventual, teorema CAP, e padrões de design para sistemas distribuídos. Os estudantes desenvolvem projetos práticos que combinam diferentes tecnologias de banco de dados para resolver problemas reais.

Conteúdos

View

Views são objetos de banco de dados que funcionam como "consultas salvas", permitindo simplificar consultas complexas, controlar o acesso a dados sensíveis e reutilizar lógicas de filtragem ou junção de tabelas. Elas são essenciais para abstrair a complexidade do esquema de banco de dados.

Function e Procedure

Functions e Procedures são blocos de código SQL armazenados diretamente no servidor do banco de dados, projetados para melhorar desempenho, reutilização de código e centralização da lógica de negócio. Eles são essenciais em sistemas com alto volume de dados ou concorrência.

Linguagem Controlada de Dados - DCL

A DCL (Linguagem de Controle de Dados) é um conjunto de comandos SQL responsáveis pelo gerenciamento de acessos e permissões em bancos de dados relacionais.
