Casos de Uso - UML

Matéria: Engenharia de Software • Data: 15/05/2025

Casos de Uso UML

Diagramas de Casos de Uso são ferramentas visuais da UML que descrevem as funcionalidades de um sistema sob a perspectiva dos usuários (atores). Eles são essenciais para capturar requisitos funcionais e modelar interações entre atores e sistema.

Elementos Básicos

  • Atores:
    • Representam papéis de usuários, hardware ou sistemas externos.
    • Símbolo: "boneco palito" com rótulo abaixo.
    • Exemplos: Cliente, Funcionário, <<system>> Sistema Integrado.
  • Casos de Uso:
    • Funcionalidades do sistema representadas por elipses com verbos no infinitivo.
    • Exemplos: Abrir Conta, Realizar Login.
  • Fronteira do Sistema: Retângulo que delimita os casos de uso internos.

Tipos de Atores

Tipo Descrição Exemplo
Primário Usuários diretos do sistema (humanos). Gerente, Cliente
Secundário Sistemas externos ou hardware. <<system>> Medidor de Radiação

Relacionamentos entre Casos de Uso

  • Inclusão (<<include>>):
    • Indica que um caso de uso sempre executa outro.
    • Exemplo: Realizar Saque inclui Registrar Movimento.
  • Extensão (<<extend>>):
    • Comportamento opcional sob condições específicas.
    • Exemplo: Realizar Login pode estender para Autorregistrar.
  • Generalização:
    • Herança de funcionalidades entre casos de uso.
    • Exemplo: Abrir Conta generaliza Abrir Conta Poupança.

Documentação de Casos de Uso

Estrutura padrão para detalhar um caso de uso:

Nome do Caso de Uso: UC01 - Abrir Conta
Ator Principal: Funcionário
Atores Secundários: Cliente
Pré-condições: Cliente aprovado para abertura de conta.
Fluxo Principal:
1. Funcionário consulta CPF do cliente.
2. Sistema valida cadastro.
3. Cliente informa senha.
4. Sistema cria conta e emite cartão.
Cenário Alternativo: Cliente não cadastrado → Executar "Gerenciar Clientes".
Restrições: Depósito mínimo de R$ 5,00.

Exemplo Prático: Sistema Bancário

  • Atores: Cliente, Funcionário, Caixa Eletrônico.
  • Casos de Uso Principais:
    • Abrir Conta
    • Realizar Saque (inclui Registrar Movimento)
    • Emitir Extrato
  • Diagrama Simplificado:
    [Cliente] --- (Realizar Saque) (Realizar Saque) .> (Registrar Movimento) : <<include>>

Boas Práticas

  • Evitar poluição visual: limitar casos de uso secundários no diagrama geral.
  • Usar estereótipos (ex: <<system>>) para destacar atores não-humanos.
  • Documentar cenários alternativos e exceções detalhadamente.

Conclusão

Diagramas de Casos de Uso são fundamentais para alinhar expectativas entre stakeholders e desenvolvedores. Eles fornecem uma visão clara das funcionalidades do sistema e servem como base para a modelagem de outros diagramas UML, como Sequência e Classes.
